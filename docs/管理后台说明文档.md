# BDCS2 游戏平台 — 管理后台说明文档

> 本文档面向运营人员，详细介绍 BDCS2 游戏平台管理后台的各项功能模块、操作流程和注意事项。

---

## 一、后台登录

### 1.1 访问地址

管理后台访问路径为 `/admin`，例如：`https://youmegame.vip/admin`

### 1.2 登录方式

管理后台使用**独立的账号密码登录**，与玩家登录系统完全隔离。

| 项目 | 说明 |
|------|------|
| 默认账号 | admin |
| 默认密码 | admin123（强烈建议通过环境变量修改） |
| 登录有效期 | 8 小时 |
| 安全机制 | 同一 IP 连续 5 次登录失败后锁定 15 分钟 |

> **安全提醒**：正式上线前务必通过环境变量 `ADMIN_ACCOUNT` 和 `ADMIN_PASSWORD` 修改默认账号密码，避免安全风险。

## 二、仪表盘（Dashboard）

登录后台后首先进入仪表盘页面，展示平台核心运营数据概览：

| 数据指标 | 说明 |
|----------|------|
| 总玩家数 | 平台注册玩家总数 |
| ROLL 房总数 | 创建的 ROLL 房总数 |
| 活跃 ROLL 房 | 当前状态为"进行中"的 ROLL 房数量 |

## 三、玩家管理

### 3.1 玩家列表

玩家列表页面展示所有注册玩家的信息，支持以下操作：

**查询与筛选：**
- 按关键词搜索（昵称、手机号）
- 按状态筛选（正常/封禁）
- 按 VIP 等级筛选
- 分页浏览（每页 15 条）

**玩家信息字段：**

| 字段 | 说明 |
|------|------|
| ID | 玩家唯一标识 |
| 昵称 | 玩家昵称 |
| 手机号 | 注册手机号 |
| 金币余额 | 当前金币数量 |
| 钻石余额 | 当前钻石数量 |
| VIP 等级 | 当前 VIP 等级 |
| 累计充值 | 历史充值总金额 |
| 身份 | player（普通玩家）/ streamer（主播）/ merchant（商户） |
| 状态 | 正常 / 封禁 |
| 注册时间 | 账号创建时间 |

### 3.2 玩家详情

点击玩家可查看详细信息，包括完整的个人资料和账户数据。

### 3.3 玩家状态管理

运营人员可对玩家执行以下操作：

| 操作 | 说明 |
|------|------|
| 封禁玩家 | 将玩家状态改为封禁，需填写封禁原因。被封禁的玩家无法登录 |
| 解封玩家 | 恢复玩家正常状态 |

### 3.4 玩家身份管理

可修改玩家的身份类型和佣金设置：

| 身份类型 | 说明 |
|----------|------|
| player | 普通玩家（默认） |
| streamer | 主播，可设置佣金比例 |
| merchant | 商户，可设置佣金比例 |

**可配置项：**
- 佣金比例（0-100%）
- 是否启用佣金
- 邀请人 ID（关联上级）

### 3.5 手动调整金币

管理员可手动增加或扣除玩家金币，操作时需填写原因。所有调整都会记录在金币流水日志中，便于审计追溯。

| 操作 | 说明 |
|------|------|
| 增加金币 | 输入正数金额和原因，金币直接到账 |
| 扣除金币 | 输入负数金额和原因，余额不足时操作失败 |

## 四、财务管理

### 4.1 财务概览

财务概览页面展示充值相关的核心数据：

| 数据指标 | 说明 |
|----------|------|
| 累计充值总额 | 所有已审批通过的充值订单总金额 |
| 今日充值额 | 今天审批通过的充值金额 |
| 累计订单数 | 已审批通过的订单总数 |
| 今日订单数 | 今天审批通过的订单数 |
| 充值排行榜 | 累计充值金额最高的玩家列表 |

### 4.2 充值订单管理

展示所有充值订单，支持按状态和关键词筛选：

**订单状态：**

| 状态 | 编码 | 说明 |
|------|------|------|
| 待审批 | 0 | 玩家提交的充值申请，等待管理员处理 |
| 已通过 | 1 | 管理员审批通过，金币已发放 |
| 已拒绝 | 2 | 管理员拒绝，需填写拒绝原因 |

**审批操作：**

- **通过订单**：点击"通过"按钮，系统自动将对应金币和钻石发放到玩家账户，同时更新玩家累计充值金额和 VIP 积分。可附加备注。
- **拒绝订单**：点击"拒绝"按钮，必须填写拒绝原因。拒绝后不发放任何金币。

> **重要提示**：审批操作不可撤销，请确认后再操作。已处理的订单不可重复操作。

### 4.3 充值档位配置

管理充值页面展示的充值档位，支持增删改查：

| 配置项 | 说明 |
|--------|------|
| 金额 | 玩家需支付的人民币金额 |
| 金币 | 充值后获得的金币数量 |
| 赠送钻石 | 额外赠送的钻石数量 |
| 标签 | 显示在充值卡片上的标签（如"热门"、"推荐"） |
| 首充标记 | 是否为首充专属档位 |
| 排序 | 显示顺序，数字越小越靠前 |

| 数量 | 该奖品的份数 |

为 ROLL 房添加机器人参与者，营造活跃氛围。机器人使用真实感昵称（如"打工人小王"、"夜猫子233"等），不易被玩家识别。

| 操作 | 说明 |

> **注意**：开奖操作不可撤销，请确认奖品和参与者信息无误后再操作。

## 九、游戏配置管理

### 9.1 游戏列表

管理所有游戏的基本配置：

| 配置项 | 说明 |
|--------|------|
| 游戏名称 | 游戏显示名称 |
| RTP（返还率） | 控制游戏整体返还比例（1-200%） |
| 最小投注 | 每局最低投注金额 |
| 最大投注 | 每局最高投注金额 |
| 启用状态 | 控制游戏是否对玩家开放 |
| 备注 | 运营备注信息 |

### 9.2 全局 RTP 调整

支持一键修改所有游戏的 RTP 值，方便统一调控平台盈利水平。

> **RTP 说明**：RTP（Return to Player）即玩家返还率。例如 RTP=96 表示玩家每投注 100 金币，平均可获得 96 金币回报。RTP 越高，玩家赢的概率越大，平台利润越低。

### 9.3 游戏排序

支持上移/下移操作，调整游戏在前端的显示顺序。

### 9.4 ROLL-X 专项配置

ROLL-X 幸运转盘有独立的详细配置：

| 配置项 | 说明 |
|--------|------|
| RTP | 返还率 |
| 最小投注额 | 每次最低投注 |
| 最大投注额 | 每次最高投注 |
| 最小倍率 | 可选的最低倍率 |
| 最大倍率 | 可选的最高倍率 |
| 启用状态 | 是否开放 |

### 9.5 ROLL-X 游戏记录

可查看所有 ROLL-X 游戏的历史记录，包括玩家 ID、投注金额、选择倍率、是否中奖、赢得金额、净收益、游戏时间等。

### 9.6 Vortex 游戏配置

Vortex 游戏有独立的配置项：

| 配置项 | 说明 |
|--------|------|
| RTP | 返还率（50-99%） |
| 最小投注额 | 每次最低投注 |
| 最大投注额 | 每次最高投注 |

客服坐席通过独立的登录入口（`/agent/login`）登录工作台。

**坐席登录信息：**

| 项目 | 说明 |
|------|------|

坐席登录后进入工作台（`/agent`），可执行以下操作：

| 功能 | 说明 |
|------|------|

## 十四、操作注意事项

### 14.1 日常运营建议

1. **每日检查**：登录后台查看财务概览，及时处理待审批的充值订单
2. **内容更新**：定期更新 Banner 和广播内容，保持平台新鲜感
3. **ROLL 房运营**：定期创建新的 ROLL 房活动，适当添加机器人营造氛围
4. **游戏参数调优**：根据平台盈利情况适当调整 RTP 值
5. **客服响应**：确保有坐席在线，及时响应玩家咨询

### 14.2 安全注意事项

1. 不要在公共网络环境下登录管理后台
2. 定期修改管理员密码
3. 审批充值订单前务必核实支付信息
4. 手动调整金币时务必填写详细原因，便于后续审计
5. 开奖操作不可撤销，操作前请仔细确认

### 14.3 数据备份

平台数据存储在云数据库中，建议定期导出关键数据（玩家列表、充值记录、金币流水等）作为备份。
